# 智能调度分组设计

日期：2026-05-24

## 目标

新增管理员可配置的分组智能调度能力：当一个目标分组在请求调度时没有状态为“正常”的账号，系统自动从指定号池分组移动账号到目标分组，然后重试当前选号。

本功能采用“请求时懒触发补号”。它不做后台巡检，不做自动回流。

## 已确认语义

- “调过去”表示真正移动：从号池分组解绑账号，再绑定到目标分组。
- 移动后的账号长期留在目标分组。即使目标分组原有账号恢复，也不会自动回到号池。
- 触发条件是目标分组在网关选号时没有状态为“正常”的账号。
- 每次触发移动管理员配置的数量 `N`，默认 `1`。
- 源号池可用账号不足 `N` 时，有几个移动几个；一个都没有时保持现有“无可用账号”错误。
- “正常状态的号”复用现有调度语义，即账号必须满足当前 `Account.IsSchedulable()` / `ListSchedulable...` 的约束。

## 配置模型

配置保存在目标分组上：

- `smart_dispatch_enabled boolean not null default false`
- `smart_dispatch_source_group_id bigint null`
- `smart_dispatch_count int not null default 1`

校验规则：

- 未启用时允许 source group 为空。
- 启用时 source group 必须存在。
- source group 不能等于目标 group。
- `smart_dispatch_count` 必须大于 0。
- 选号时仍按目标请求上下文筛选候选账号，避免把不兼容账号移动到目标组后仍无法服务请求。

## 后端设计

新增一个边界清晰的智能调度服务，例如 `SmartDispatchService`，负责：

1. 接收目标分组、请求模型、平台上下文、排除账号集合。
2. 检查目标分组是否启用智能调度。
3. 在事务/锁保护下重新确认目标分组仍然没有状态为“正常”的账号。
4. 从源号池挑选最多 `N` 个候选账号。
5. 原子移动账号：删除源组绑定，新增目标组绑定。
6. 发送调度缓存/快照失效事件。
7. 返回移动数量和账号 ID，供日志记录和后续重试。

网关选择链路只在“目标组账号列表为空或筛选后没有候选账号”时调用智能调度服务。成功移动账号后，当前请求只重试一次正常选号，避免错误配置导致递归重试。

## 候选账号规则

源号池候选账号必须满足：

- 当前状态正常。
- 兼容目标分组平台和当前请求平台逻辑。
- 如果请求带模型，账号必须支持该模型。
- 不在当前请求失败排除列表里。
- 智能调度只按号池分组里的正常账号补号，不再按当前请求模型额外过滤。

候选账号排序沿用现有分组调度排序：先按 `account_groups.priority`，再按账号 `priority`。这样管理员在号池中设置的优先级仍然有效。

## 并发与一致性

补号动作必须防止并发重复移动：

- 对目标分组使用事务级锁或等价的 DB 锁。
- 锁内重新查询目标分组正常账号数。
- 如果其他请求已经补过号，本次不再移动。
- 移动账号时使用同一事务删除源组绑定并新增目标组绑定。

移动完成后必须失效调度缓存/快照：

- 对移动账号发账号分组变更事件。
- 对源分组和目标分组发分组变更事件或等价 payload。

这样可以避免 scheduler snapshot 继续使用旧分组关系。

## 管理端设计

在管理员分组创建/编辑表单新增“智能调度”区块：

- 开关：启用智能调度。
- 号池分组下拉：选择源分组，不允许选择当前目标分组。
- 数量输入：无正常账号时移动几个账号，默认 `1`。

后端接口、DTO、前端类型同步增加字段。普通用户分组接口不返回智能调度配置，只在管理员接口返回。

## 错误处理

- 未启用智能调度：保持现有行为。
- 配置无效：保存分组时拒绝。
- 运行时源号池为空或无兼容账号：不修改分组关系，返回现有无可用账号错误。
- 移动部分成功不允许发生；移动操作必须事务化。
- 重试选号仍失败：返回现有错误，并记录智能调度已移动的账号 ID，方便排查账号模型兼容、额度或并发问题。

## 测试计划

后端：

- 未开启智能调度时无可用账号保持原行为。
- 目标组无正常账号时从源号池移动 `N` 个。
- 移动后源组解绑、目标组绑定。
- 源组不足 `N` 个时只移动可用数量。
- 源组无候选账号时不修改绑定并返回无可用账号。
- 并发触发时只补一次。
- 请求带 excluded account 时不会移动已排除账号。
- 分组创建/更新校验 source group、count 和自引用。
- DTO 只在管理员接口暴露智能调度配置。

前端：

- 分组创建/编辑表单可以保存智能调度配置。
- 编辑已有分组时能正确回填开关、源分组和数量。
- 禁止选择当前分组作为号池。
- 关闭开关时不强制选择号池。

## 非目标

- 不做后台定时巡检。
- 不做自动回流。
- 不做跨目标组配额分配策略。
- 不改变现有账号状态定义。
- 不改变现有用户 API Key 绑定分组权限语义。
